# トモシビモンスターズ

ブラウザで遊べる、オリジナルモンスター育成RPGです。
モンスターを集めて育て、ジムをめぐり、シャドウ団を止めて、チャンピオンをめざす本編ストーリーに加えて、
**他の人との「こうかん」「たいせん」を Supabase の通信でおこなえます。**

外部ライブラリのビルドは不要。純粋な HTML + JavaScript(ES Modules)+ Canvas だけで動きます。

---

## あそびかた(すぐ始める)

Node.js が入っていれば、フォルダ内で:

```bash
node server.js
```

を実行し、ブラウザで **http://localhost:8788** を開くだけです。

> Node を使わず、VS Code の Live Server など任意の静的サーバーで `index.html` を開いても動きます。
> (ES Modules を使うため、`file://` で直接開くのは不可。かならずサーバー経由で開いてください)

### 操作方法

| キー | 動作 |
|------|------|
| 矢印キー / WASD | 移動・カーソル |
| Z / Enter / Space | 決定・話す・調べる |
| X / Esc | キャンセル・メニューを開く |
| Shift(押しっぱなし) | ダッシュ |
| M | BGM/効果音のミュート切替 |

## ゲームの流れ(ストーリー)

トモシビ地方の「ハジメむら」から冒険が始まります。

1. **カエデ研究所**で最初のパートナー(リーフィ / ヒノコン / シズミン)を選ぶ
2. 草むらで野生モンスターを捕まえ、トレーナーと戦いながら旅をする
3. **4つのジム**(むし → みず → でんき → ゴースト)を攻略してバッジを集める
4. 各地で暗躍する **シャドウ団** を止める(発電所・港・アジト)
5. 伝説のドラゴン **テンゲンリュウ** をめぐる「ほしふりの とう」
6. **チャンピオンロード → トモシビでんどう** でチャンピオンに挑戦
7. ライバル「ソラ」との3度の対決

- モンスターは全 **27種**、レベルアップで **進化**・技習得。
- タイプ相性・状態異常(どく/まひ/やけど/ねむり)・急所・能力ランク変化あり。
- 「かいふくセンター」で回復、「ショップ」で買い物、パソコンで **ボックス** 管理。
- セーブはメニューの「レポート」から(ブラウザの localStorage に保存)。

## 通信機能(こうかん・たいせん)のセットアップ

通信は **Supabase の Realtime(broadcast チャンネル)** を使います。
**データベースのテーブルや SQL は不要**で、無料プランのままで動きます。

> 設定しなくても本編はすべて遊べます。設定すると、メニューの「つうしん」が有効になります。

### 手順

1. https://supabase.com にログインし、**新しいプロジェクト**を作成(無料でOK)。
2. プロジェクトのダッシュボード左下 **Settings → API** を開く。
3. 次の2つをコピー:
   - **Project URL**(例: `https://xxxxxxxx.supabase.co`)
   - **Project API keys** の **`anon` `public`** キー
4. このフォルダの **`netconfig.js`** を開き、貼りつけて保存:

```js
export const NET_CONFIG = {
  SUPABASE_URL: "https://xxxxxxxx.supabase.co",
  SUPABASE_ANON_KEY: "eyJhbGciOiJIUzI1NiIs...(anonキー)...",
};
```

5. ページを再読み込み。メニューの「つうしん」から遊べます。

> Realtime は新規プロジェクトでは既定で有効です。もし通信できない場合は
> Supabase の **Database → Replication** や **Realtime** の設定が有効か確認してください
> (broadcast チャンネルのみ使用するため、通常は追加設定不要です)。

### あそびかた(2人で)

1. 2人ともゲームを開き、メニュー →「つうしん」へ。
2. 「モンスターこうかん」か「つうしんたいせん」を選ぶ(**2人とも同じほうを選ぶ**)。
3. 片方が **「ルームを つくる」** を選ぶと **4桁のあいことば** が表示される。
4. もう片方が **「あいことばで はいる」** を選び、その4桁を入力する。
5. つながったら:
   - **こうかん**: おたがい出すモンスターを選び、両者が「はい」で成立。
   - **たいせん**: おたがいの手持ちを**満タンのコピー**で6vs6のフルバトル(手持ちは減りません)。

> あいことばは同じ Supabase プロジェクトを使う人どうしで有効です。
> 友達には `netconfig.js` を共有するか、同じ URL/anonキーを使ってもらってください。

## ファイル構成

```
index.html            エントリ(canvasとスクリプト読み込み)
server.js             依存ゼロの静的サーバー(node server.js)
netconfig.js          ★通信の設定(Supabase URL / anonキー)
css/style.css         画面まわりのスタイル
js/
  main.js             起動
  game.js             ゲームループ・シーンスタック・プレイヤー状態
  core/
    monster.js        個体生成・ステータス・経験値・進化・シリアライズ
    battle.js         バトル計算エンジン(UI非依存・イベント列を返す)
    save.js           セーブ/ロード(localStorage)
  engine/
    input.js          キーボード入力
    audio.js          WebAudio 効果音・簡易BGM
    render.js         タイル・スプライト・テキスト・ウィンドウ描画
    ui.js             メッセージ送り・選択メニュー(Promise API)
  data/
    types.js          タイプと相性表
    moves.js          技データ
    dex.js            モンスター図鑑(種族値・進化・習得技)
    sprites.js        ドット絵(モンスター・人物)
    items.js          道具データ
    maps.js           全マップ(村・道路・洞窟・ジム・塔 …)
    events.js         ストーリーイベントのスクリプト
  net/
    net.js            Supabase Realtime ラッパー(ルームコード通信)
  scenes/
    title.js  overworld.js  battle_scene.js  menu.js  box.js  shop.js  link.js
tools/
  validate.mjs        データ整合性チェック
  simbattle.mjs       バトルエンジンの自動テスト
```

## 開発者向けメモ

データ検証とバトルの自動テスト:

```bash
node tools/validate.mjs    # マップ/図鑑/技/スプライトの整合性チェック
node tools/simbattle.mjs   # ランダム対戦を大量に流して例外・不正値を検出
```

- モンスターやマップを追加したら `validate.mjs` を通すと安全です。
- バトルロジック(`core/battle.js`)は描画から独立していて、「イベント列」を返すだけなので、
  ローカル戦・通信戦(ホスト/ゲスト)で同じコードを共有しています。
- 通信対戦はホスト側だけが計算し、結果イベントをゲストへ送る方式(決定性ズレを防止)。

たのしんでください!
